# 💸 Мои финансы

Личный учёт денег: остаток на счёте Т-Банка, наличные в разных валютах
с пересчётом по живому курсу ЦБ РФ и ручные операции с категориями.
Статический сайт без бэкенда — хостится на GitHub Pages, открывается с
телефона. История из банковской выписки снесена (в «Справке о движении
средств» нет оплат по СБП, так что она всё равно неполная) — при желании
её можно вернуть конвейером из `scripts/`.

Карточка «Мои деньги»: счёт + наличные по валютам + итого в ₽. Курс —
cbr-xml-daily.ru (зеркало ЦБ РФ), запасной источник open.er-api.com,
кэшируется на 6 часов. Ручная операция списывается со счёта или с
наличных выбранной валюты; остатки правятся кнопкой «✎ Остатки».
Карточные траты округляются вверх до кратных 100 ₽ (как в банке):
в тратах — реальная сумма, со счёта уходит округлённая, разница
показана строкой «Ушло в копилку округлениями».

## Синхронизация устройств

Кнопка ⟳ в шапке. Зашифрованный слепок всего пользовательского состояния
(операции, категории, остатки) коммитится в ветку `sync-state` этого же
репозитория (файл `state.enc.json`) и читается другими устройствами;
побеждает последняя запись. Нужен токен GitHub со scope `repo` — один раз
на устройство, хранится в localStorage. Расшифровать слепок можно только
паролем от сайта. Настраивайте сначала устройство с данными, потом пустые.

## Что умеет

- помесячный график трат по категориям, KPI-плитки, поиск и фильтры
  (период, тип операций, категория);
- автокатегоризация по правилам-подстрокам (`scripts/categorize.py`);
- **округление в копилку**: банк округляет каждую покупку вверх до
  кратных 100 ₽ и разницу переводит в Инвесткопилку — парсер связывает
  эти пары (784/784 нашлись), галка под KPI показывает траты «как
  списалось со счёта» либо «как в чеке»;
- **сверка баланса**: плитка «На счету» считается от реального остатка
  (задаётся в `scripts/categorize.py` третьим аргументом) — видно, что
  все траты учтены;
- **разбор переводов**: переводы не категоризируются автоматически —
  баннер предлагает раскидать их руками, по одному или сразу все с
  одинаковым описанием. Категория «Наличные (валюта)» спрашивает, в
  какую сумму какой валюты конвертировался перевод;
- **ручные операции**: «＋ Операция» — траты и поступления мимо счёта
  (наличные и т.п.), на сверку баланса не влияют;
- **возмещения**: плюсовой перевод можно назначить в категорию трат
  (друзья скинули за каршеринг/еду) — он вычтется из суммы трат и из
  этой категории; возвраты покупок учитываются так же автоматически;
- **редактирование категорий**: ✎ на любой категории — имя, эмодзи,
  цвет, тип (трата/доход/перевод), свои правила поверх встроенных;
  категории «внутреннего перекладывания» (копилка, кредитка) делайте
  типом «Переводы», чтобы они не попадали ни в траты, ни в доходы;
- свои категории: «＋ Своя категория» или из карточки операции — имя,
  эмодзи, цвет, правила. Всё пользовательское (категории, правки,
  валюты, ручные операции) хранится в localStorage браузера;
- выгрузка отфильтрованных операций в CSV (`операции.csv`), плюс готовая
  таблица `transactions.csv` в корне репозитория.

## Шифрование

Репозиторий публичный (условие бесплатного GitHub Pages), но операции в
открытом виде в него не попадают: в git лежит только `data.enc.js` —
AES-256-GCM, ключ из пароля через PBKDF2 (600 000 итераций SHA-256).
Сайт при открытии спрашивает пароль и расшифровывает данные прямо в
браузере; пароль можно запомнить на устройстве или передать один раз в
ссылке как `…/#p=пароль` (из адресной строки он сразу затирается).

## Файлы

| Файл | Что это |
|---|---|
| `index.html`, `style.css`, `app.js` | сайт |
| `data.enc.js` | зашифрованные операции + базовые категории |
| `scripts/parse_statement.py` | парсер PDF-выписки (pdfplumber), сверяет суммы с итогами банка |
| `scripts/categorize.py` | категории, правила, санитизация → `data.js` + `transactions.csv` (локально, в git не попадают) |
| `scripts/encrypt_data.mjs` | `data.js` → `data.enc.js` |

## Обновить данные по новой выписке (или сменить пароль)

```bash
pip install pdfplumber
python3 scripts/parse_statement.py /tmp/transactions.json   # путь к PDF внутри скрипта
python3 scripts/categorize.py /tmp/transactions.json data.js
node scripts/encrypt_data.mjs data.js data.enc.js "новый-пароль"
git add data.enc.js && git commit -m "Обновление данных" && git push
```

Сам PDF, `data.js` и `transactions.csv` в репозиторий не кладутся
(`.gitignore`). Номера телефонов и счетов в описаниях замаскированы ещё
на этапе генерации.
